## Project Overview

**Boom Tap 2 Player** — A Unity 2022.3.62f1 (LTS) casual mobile game for Android, built with C# and exported via IL2CPP. Package ID: `com.codegym.Boomchip2player`. Targets Android 7.0+ (API 24), compiles against API 34.

## Build & Run

This is a Unity project. The primary workflow is through the Unity Editor, not Android Studio directly.

- **Open in Unity:** Use Unity Hub → Open Project (requires Unity 2022.3.62f2)
- **Android Build:** Unity menu → File → Build Settings → Android → Build
- **Gradle location:** `Library/Bee/Android/Prj/IL2CPP/Gradle/` (auto-generated by Unity, do not edit directly)
- **Supported ABIs:** armeabi-v7a, arm64-v8a
- **MultiDex:** Enabled
- **Testing:** Unity Test Framework 1.1.33 is installed but no custom tests exist yet. Run via Unity Test Runner window.

## Architecture

**Singleton Manager Pattern** — Core systems are MonoBehaviour singletons with `DontDestroyOnLoad`.

### App Initialization Flow
1. **Loading Scene** → `LoadingManager` initializes Firebase, AppsFlyer, loads Remote Config
2. **Gameplay Scene** → `MenuManager` handles main menu and mode selection
3. **Game Mode Scenes** → Each mode has its own manager (`BoomChipManager`, `Mode2Manager`, `Mode3Manager`, `DiceModeManager`)

### Core Managers (persist across scenes)
- `AdsManager` — AdMob lifecycle: banner, interstitial, rewarded, app open, MREC ads
- `FirebaseManager` — Firebase init, analytics event logging
- `AppsflyerManager` — AppsFlyer SDK init and event tracking
- `AudioManager` — Sound/music playback
- `SettingManager` — Settings UI and persistence
- `UnlockSystemManager` — Feature unlock logic via Remote Config + rewarded ads

### Static Utilities
- `GlobalSettings` — Centralized settings state (music volume, vibration toggle)
- `AdEventTracker` — Unified analytics dispatcher (Firebase + AppsFlyer dual-tracking)

### Persistence
- Uses `PlayerPrefs` for local data storage (no database)

## Game Modes

1. **Boom Chip** (main) — 2-player tile prediction game with coin flip. Scripts in `Assets/Scripts/Gameplay/Boom cheat/`
2. **Will You Marry** — Prediction/challenge mode. `Mode2Manager.cs`
3. **Bounce** — Item catching with drag controls. Scripts in `Assets/Scripts/Gameplay/Mode3GamePlay/`
4. **Dice** — RNG prediction. Scripts in `Assets/Scripts/Gameplay/BoomDice/`

## Key Scripts Directory: `Assets/Scripts/`

```
Ads/           — Ad management, unlock system, debug tools

Gameplay/      — Menu, profiles, all 4 game mode implementations
```

## Advertising Scenarios

All ad placements are controlled via Firebase Remote Config (default: FALSE/off). Ad revenue is dual-tracked to Firebase Analytics and AppsFlyer.

### Quy tắc chung khi triển khai ads
1. **Preload:** Luôn preload ad tiếp theo sau khi ad hiện tại được hiện/đóng.
2. **Báo lỗi network:** Nếu ad load thất bại do lỗi network → hiển thị trang yêu cầu người dùng kiểm tra kết nối mạng rồi thử lại. Nếu lỗi khác (không phải network) → cho người dùng tiếp tục bình thường.
3. **Không double ads:** Không hiện 2 fullscreen ad liên tiếp mà không có user interaction ở giữa (ngoại trừ native-after-inter là có chủ đích).
4. **Remote Config first:** Luôn fetch và activate RemoteConfig khi app start trước khi hiện bất kỳ ad nào.
5. **Test IDs:** Dùng Google AdMob test ad unit IDs khi phát triển. KHÔNG dùng real IDs trong debug builds.

### App Open Ad (AOA) — FullScreen
| Trigger | Remote Config Key | Note |
|---------|-------------------|------|
| App launch | `show_open_ads` (bool, default FALSE) | Không show lần đầu tiên, từ lần thứ 2 trở đi mới show |
| App launch (first open) | `show_open_ads_first_open` (bool, default FALSE) | Kiểm soát riêng việc show AOA lần đầu |
| Resume from background | `show_resume_ads` (bool, default FALSE) | Khi user quay lại app từ home, app khác, lock/unlock |

### Interstitial — FullScreen
| Trigger | Remote Config Key | Note |
|---------|-------------------|------|
| Player 1 chọn xong 3 item → Next | `is_show_inter_p1_choose` (bool) | Áp dụng tất cả mode chơi |
| Player 2 chọn xong 3 item → Next | `is_show_inter_p2_choose` (bool) | Áp dụng tất cả mode chơi |
| Back về home | `is_show_inter_back_home` (bool) | Tất cả vị trí back home, icon home |
| Retry | `is_show_inter_retry` (bool) | Tất cả vị trí retry |

**Interval:** `ads_interval` (number, default 45s) — khoảng cách tối thiểu giữa 2 lần show interstitial.

### Banner — Bottom of Screen
| Remote Config Key | Type | Note |
|-------------------|------|------|
| `is_show_banner` | bool, default FALSE | Bật/tắt banner trên tất cả màn hình |
| `time_reload_collap_ad` | number, default 10 | Thời gian (giây) reload collapsible banner. Không điền = không show collap banner |

Banner hiển thị ở tất cả màn hình, hỗ trợ định dạng collapsible.

### MREC — Bottom, phía trên Banner
| Screen | Remote Config Key | Note |
|--------|-------------------|------|
| Player 1 Choose | `is_show_mrec_p1_choose` (bool) | Màn hình chọn 3 item |
| Player 2 Choose | `is_show_mrec_p2_choose` (bool) | Màn hình chọn 3 item |
| Loading Game | `is_show_mrec_loading_game` (bool) | Sau khi cả 2 player chọn xong, vào loading |
| Gameplay | `is_show_mrec_gameplay` (bool) | Chỉ áp dụng ở Mystery Boom và Roll & Collect |
| Complete Game | `is_show_mrec_complete_game` (bool) | Tất cả màn hình complete ở mọi mode |

### Rewarded Ad — FullScreen
| Trigger | Remote Config Key | Note |
|---------|-------------------|------|
| Unlock Challenge mode | `is_show_rw_challenge` (bool) | Xem reward để unlock |
| Unlock Prediction mode | `is_show_rw_prediction` (bool) | Xem reward để unlock |
| Unlock Avatar | `is_show_rw_profile` (string) | Value: danh sách avatar ID (vd: `"1,2,3,4"`). Unlock vĩnh viễn sau khi xem |

### Other Remote Config
| Key | Type | Default | Note |
|-----|------|---------|------|
| `rating_popup` | bool | FALSE | Hiển thị In-App Review dialog (Google Play Core) |
